# claude-chat-to-markdown

Claude.ai 専用の Chrome 拡張機能。会話ログとセッションのファイルを保全し、Ctrl+Enter による誤送信防止を提供します。

会話を Markdown として手元に残し、Claude が生成したファイルを ZIP にまとめて保全することを目的としています。会話ログとしての記録、スレッドとしての証跡保存に使えます。

## 機能

### 保存パネル（画面右に表示されるドラッグ可能なパネル）

- **MD**（緑）
  ボタンを押した時点で DOM を一度スキャンし、会話ログを Markdown で保存します。
  User / Model / 添付ファイル名を時系列順に記録します。

- **MD+ZIP**（青）
  押すと以下を一括で実行します。
  1. 会話ログ Markdown を保存（`AI_claude_ai_<id>_<ts>.md`）
  2. 入力欄に ZIP 生成プロンプトを自動入力・送信し、Claude に 2 つの ZIP を作らせます
     - `..._uploads.zip` … アップロード/ペーストしたファイルの実体
     - `..._downloads.zip` … Claude がそのセッションで生成したファイル

  > **注意：** アップロードファイルの内容は DOM に展開されないため、内包 MD の自動生成は行っていません。アップロードしたファイルの実体保全は Claude に依頼する `..._uploads.zip` で行います。今後のバージョンで対応予定です。

- **ガード: ON/OFF**（紫/グレー）
  誤送信防止ガードの有効・無効を切り替えます。状態は保存されます。

### 誤送信防止ガード（enter-guard）

画面上部に表示されるバッジで状態を確認できます（ドラッグで移動可能）。

- **Ctrl+Enter** … 入力末尾に境界封印 `::role_User:<timestamp>` を注入します。送信はしません。
- **通常の Enter** … 封印がない場合は送信をブロックします。封印がある場合のみ送信を許可します。

これにより「Ctrl+Enter で封印 → Enter で送信」という 2 段階の操作が必要になり、書きかけの誤送信を防ぎます。注入されたマーカーは会話保存時に User 発言の境界としても機能します。

| バッジ表示 | 状態 |
| --- | --- |
| 🔒 GUARD ACTIVE | ガード有効・未封印 |
| ✅ SEALED (Press ENTER to Send) | 封印済み・送信可能 |
| ⚠️ SEAL REQUIRED (Ctrl+Enter) | 封印なしで Enter を押した |
| ⚪ GUARD OFF | ガード無効 |

## 出力ファイルの命名

すべて以下の規則で統一されます。

```
AI_claude_ai_<チャットID>_<unixtime>_<YYYYMMDDhhmm>.md            … 会話ログ
AI_claude_ai_<チャットID>_<unixtime>_<YYYYMMDDhhmm>_uploads.zip   … アップロード/ペースト実体
AI_claude_ai_<チャットID>_<unixtime>_<YYYYMMDDhhmm>_downloads.zip … Claude生成物
```

同一プレフィックスで並ぶため、どのファイルがどのセッションのものか一目で対応が取れます。

## インストール

1. このリポジトリを ZIP でダウンロードするか `git clone` します。
2. Chrome で `chrome://extensions` を開きます。
3. 右上の「デベロッパーモード」を有効にします。
4. 「パッケージ化されていない拡張機能を読み込む」をクリックし、`claude-chat-to-markdown` フォルダを選択します。

## 使い方

1. claude.ai でセッションを開きます。
2. 拡張機能のアイコンをクリックすると保存パネルの表示/非表示が切り替わります。
3. 保存したいタイミングで **MD** または **MD+ZIP** を押します。
4. **MD+ZIP** の場合、ZIP 生成プロンプトが自動送信されるので、Claude が出力した「すべてダウンロード」から 2 つの ZIP を取得します。

## 制約

- 添付ファイルの中身はサーバ側にあり、DOM に展開されません。アップロードファイルの内包 MD は現バージョンでは生成しません（今後対応予定）。実体は `..._uploads.zip` に保全されます。
- Claude が生成したファイルの実体も DOM には残らないため、`..._downloads.zip` は Claude 自身に生成させる方式を取っています。
- 自動送信は Claude の入力欄・送信ボタンの DOM 構造に依存します。UI 変更時は動作しなくなる可能性があります。

## 権限

- `storage` … パネル表示状態・ガード状態・バッジ位置の保存
- `tabs` … アクティブタブへのメッセージ送信
- `downloads` … ファイルのダウンロード
