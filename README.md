# claude-chat-to-markdown

Claude.ai 専用の Chrome 拡張機能。会話を逐語で記録し、扱ったファイルの中身まで埋め込んだ「1ファイルで完結する正確なログ」を作ることを目的としています。Ctrl+Enter による誤送信防止も備えます。

LLM の記憶は要約・圧縮されるため、経緯を正確に引き継ぐには逐語に近い外部ファイルを参照させるのが有効です。この拡張はそのためのログを生成します。

## 機能

### 保存パネル（画面右に表示されるドラッグ可能なパネル）

- **MD**（緑）
  押した時点で DOM を一度スキャンし、会話ログを逐語の Markdown で保存します。User / Model / 添付ファイル名を時系列順に記録します。拡張機能だけで完結します。

- **MD+ZIP**（青）
  1 回押すと以下を実行します。
  1. 逐語の会話 Markdown を保存（`..._<ts>.md`）
  2. 「逐語の会話 ＋ 生 DOM（照合用）＋ ファイル名一覧 ＋ 指示」を JSON にまとめ、Claude の入力欄へ自動アップロード
  3. プロンプトを自動送信し、Claude に次の 3 つを生成させます
     - **記録版 Markdown**（`..._record.md`）… 逐語の対話に加え、扱った各ファイルの**中身をコードブロックで埋め込んだ自己完結ログ**。これ 1 ファイルで対話とファイル内容が完結します。
     - `..._uploads.zip` … アップロード/ペーストしたファイルの実体
     - `..._downloads.zip` … Claude がそのセッションで生成したファイルの実体

  記録版は「要約・改変をせず逐語のまま、生 DOM で取りこぼしを補完」するよう指示されます。テキストはそのまま、200KB 超は先頭 10 行、zip は展開してテキストのみ、mhtml は除外します。

- **最初に**（青緑）
  新規チャットの冒頭で押すと、「これからアップロードするファイルは同名でもバージョン番号を付けて別個に扱い、保存名を応答で明示して」という依頼を Claude に送ります。同名ファイルの上書きを避け、後の記録と整合を取るための準備です。

- **ガード: ON/OFF**（紫/グレー）
  誤送信防止ガードの有効・無効を切り替えます。状態は保存されます。

### 誤送信防止ガード（enter-guard）

画面上部のバッジで状態を確認できます（ドラッグで移動可能）。

- **Ctrl+Enter** … 入力末尾に境界封印 `::role_User:<timestamp>` を注入します（送信はしません）。
- **通常の Enter** … 封印がなければ送信をブロック、封印があれば送信を許可します。

「Ctrl+Enter で封印 → Enter で送信」の 2 段階により、書きかけの誤送信を防ぎます。注入マーカーは会話保存時に User 発言の境界としても機能します。

| バッジ表示 | 状態 |
| --- | --- |
| 🔒 GUARD ACTIVE | ガード有効・未封印 |
| ✅ SEALED (Press ENTER to Send) | 封印済み・送信可能 |
| ⚠️ SEAL REQUIRED (Ctrl+Enter) | 封印なしで Enter を押した |
| ⚪ GUARD OFF | ガード無効 |

## 出力ファイルの命名

```
AI_claude_ai_<チャットID>_<unixtime>_<YYYYMMDDhhmm>.md            … 逐語の会話ログ（拡張機能が生成）
AI_claude_ai_<...>_record.md                                      … 記録版（Claudeが生成、ファイル内容埋め込み）
AI_claude_ai_<...>_uploads.zip                                    … アップロード/ペースト実体
AI_claude_ai_<...>_downloads.zip                                  … Claude生成物
```

同一プレフィックスで並ぶため、どのファイルがどのセッションのものか一目で対応が取れます。

## インストール

1. このリポジトリを ZIP でダウンロードするか `git clone` します。
2. Chrome で `chrome://extensions` を開きます。
3. 「デベロッパーモード」を有効にします。
4. 「パッケージ化されていない拡張機能を読み込む」をクリックし、フォルダを選択します。

## 使い方

1. claude.ai でセッションを開きます。
2. 拡張機能アイコンでパネルの表示/非表示を切り替えます。
3. 新規チャットなら最初に **最初に** を押してファイル同期を依頼しておくと、同名ファイルの追跡が楽になります。
4. 記録したいタイミングで **MD**（逐語ログのみ）または **MD+ZIP**（記録版＋ZIP一式）を押します。
5. **MD+ZIP** では Claude が生成した記録版 MD と 2 つの ZIP を「すべてダウンロード」から取得します。

## 仕組みと制約

- 会話の逐語抽出は拡張機能が行い、ファイル内容の埋め込みは Claude が行います。記録版には逐語の会話に加えて生 DOM も渡し、Claude が取りこぼしを照合できるようにしています。
- アップロード添付・生成ファイルの実体は DOM に残らないため、ZIP は Claude 自身に生成させます。
- 自動アップロード・自動送信は Claude の入力欄・送信ボタン・ファイル入力欄の DOM 構造に依存します。UI 変更時は動作しなくなる可能性があります。
- `..._record.md` の品質は Claude の応答に依存します。逐語性を重視する指示を入れていますが、確認は利用者側で行ってください。

## 権限

- `storage` … パネル表示状態・ガード状態・バッジ位置の保存
- `tabs` … アクティブタブへのメッセージ送信
- `downloads` … ファイルのダウンロード
